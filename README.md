# 칩 카운터

친구들과 포커 칠 때 카드는 직접, 칩 계산만 폰으로 실시간 공유하는 웹앱입니다.

## 로컬에서 실행

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속.

## Render에 배포하기

1. 이 폴더 전체를 GitHub 저장소에 올립니다 (private repo도 가능).
2. [Render](https://render.com) 가입 후 **New + → Web Service** 선택.
3. 방금 만든 GitHub 저장소를 연결합니다.
4. 설정값:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. **Create Web Service** 클릭 → 배포가 끝나면 `https://your-app-name.onrender.com` 같은 주소가 생깁니다.
6. 이 주소를 친구들에게 카톡 등으로 공유하면, 각자 폰 브라우저에서 접속만 하면 됩니다.

> 무료 플랜은 일정 시간 미사용 시 서버가 슬립 모드로 들어갑니다. 게임 시작 전 누군가 먼저 한 번 접속해서 서버를 깨워두면 좋습니다 (슬립 상태에서 첫 접속은 10~30초 정도 걸릴 수 있어요).

## 사용 방법

1. **새 게임 만들기**를 누르면 4자리 방 코드가 자동 생성됩니다.
2. 생성된 화면에서 "링크 복사"를 눌러 친구들에게 공유하세요. 친구들은 그 링크로 들어오면 자동으로 같은 방에 입장합니다. (또는 방 코드를 알려주고 직접 "방 코드로 입장"에 입력해도 됩니다.)
3. "바이인 설정"에서 시작 칩 금액을 입력하고 전원 칩을 리셋합니다.
4. 플레이어 이름을 추가합니다.
5. 핸드가 끝나면 이긴 사람 칩에 +, 진 사람은 -로 조정합니다 (빠른 버튼 100/500 또는 직접 입력).
6. 베팅 중인 팟을 추적하려면 상단 "팟에 추가"로 기록하고, 가져가는 사람에게 칩을 더해준 뒤 "팟 비우기"를 누릅니다.
7. 게임이 끝나면 "정산하기"를 눌러 누가 누구에게 얼마를 줘야 하는지 최소 거래 횟수로 자동 계산된 결과를 확인합니다.

## 데이터 보존에 대해

게임 데이터는 서버 메모리에만 저장됩니다. 서버가 재시작되거나(예: 무료 플랜 슬립 후 재기동) 모든 인원이 30분 이상 접속을 끊으면 해당 방의 데이터는 사라집니다. 한 게임이 끝나면 다시 쓰지 않는 용도에 맞춘 설계입니다.
