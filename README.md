# 정현이의 토큰 계산 사이트

친구들과 포커 칠 때 카드는 직접, 토큰(칩) 계산만 폰으로 실시간 공유하는 웹앱입니다.
방장이 방을 만들고, 순서대로 베팅하고, 방장이 승자를 지정하는 방식으로 동작합니다.

## 동작 방식

- **방 만들기**: 방장이 이름, 인당 보유 금액(시작 토큰), 입장료(ante)를 설정합니다.
- **입장**: 누구든 방 코드로 입장하는 순간(게임 시작 전이든 중간 참가든) 보유 금액을 받고, 즉시 입장료가 차감되어 팟에 들어갑니다.
- **매 판 입장료**: 방장이 승자를 지정해서 한 판이 끝날 때마다, 파산하지 않은 모든 플레이어에게 다시 입장료가 걷힙니다.
- **턴 순서**: 입장한 순서대로 베팅 차례가 정해집니다. 한 번에 한 사람만 베팅할 수 있고, 화면 상단에 지금 누구 차례인지 크게 표시됩니다. 잔액이 0원이라 베팅할 수 없는 사람(파산자는 아님)의 차례는 자동으로 다음 사람에게 넘어갑니다.
- **베팅**: 본인 차례일 때만 베팅 버튼(100 / 500 / 1000 / 올인)이 보입니다. 버튼을 누르면 일단 "베팅 중인 금액"으로 쌓이기만 하고 아직 확정되지 않습니다.
  - **초기화**: 확정 전이라면 쌓인 베팅액을 0으로 되돌릴 수 있습니다 (예: 500을 눌렀는데 200을 원했다면 초기화 후 100을 두 번).
  - **베팅 확정**: 누르면 그 금액이 실제로 잔액에서 빠지고 팟에 들어가며, 다음 사람 차례로 넘어갑니다.
  - **체크**: 베팅 중인 금액이 0일 때는 "체크" 버튼이 대신 나타납니다. 누르면 베팅 없이 그대로 다음 사람 차례로 넘어갑니다.
  - **다이**: 본인 차례에 "다이" 버튼을 누르면 그 판에서 빠집니다. 이미 베팅했던 금액은 그대로 팟에 남고 돌려받지 않습니다. 다이한 사람은 그 판이 끝날 때까지 턴에서 제외되고, 방장의 승자 지정 후보에도 나오지 않습니다. 다음 판이 시작되면 다시 정상적으로 참여합니다.
- **파산의 정의**: 잔액이 0원이 되는 것 자체는 파산이 아닙니다(올인했거나 입장료를 내서 0원이 된 경우 모두 정상 상태로 취급됩니다). **다음 판 입장료를 낼 돈이 없을 때만 파산** 처리되어 "파산" 표시가 붙고 턴 순서에서 완전히 제외됩니다.
- **한 바퀴 순환**: 마지막 사람까지 베팅하면 자동으로 다시 첫 번째 사람부터 턴이 돌아갑니다.
- **승자 지정**: 카드 게임은 오프라인에서 진행하고, 끝나면 방장만 "승자 지정하기"로 한 명을 선택할 수 있습니다. 팟 전액이 그 사람에게 지급되고, 곧바로 다음 판 입장료가 전원에게 걷히며(이때 못 내는 사람은 파산 처리), 팟은 그 입장료들로 다시 채워집니다. **다음 판은 승자부터 턴이 시작**됩니다.

권한은 모두 서버에서 검증합니다 — 본인 차례가 아니면 베팅 자체가 거부되고, 방장이 아니면 승자 지정 요청이 거부됩니다.

## 로컬에서 실행

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속.

## Render에 배포하기

1. 이 폴더 전체를 GitHub 저장소에 올립니다.
2. [Render](https://render.com) 가입 후 **New + → Web Service** 선택.
3. 만든 GitHub 저장소를 연결합니다.
4. 설정값:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. **Create Web Service** 클릭 → 배포 완료되면 `https://your-app-name.onrender.com` 같은 주소가 생깁니다.
6. 이후 GitHub에 새로 push하면 Render가 자동으로 재배포합니다 (Auto-Deploy 기본 켜짐).

> 무료 플랜은 일정 시간 미사용 시 서버가 슬립 모드로 들어갑니다. 게임 시작 전 누군가 먼저 한 번 접속해서 서버를 깨워두면 좋습니다.

## 사용 방법

1. 방장이 **"방 만들기"** 탭에서 이름, 인당 보유 금액(예: 1000), 입장료(예: 100)를 입력하고 방을 만듭니다. 방장도 입장과 동시에 입장료가 빠집니다.
2. **"링크 복사"**로 방 링크를 친구들에게 공유합니다.
3. 친구들이 입장하면 보유 금액을 받고 즉시 입장료가 빠지며, 입장한 순서대로 턴 순서에 줄을 섭니다.
4. 화면 상단에 표시되는 "현재 차례"를 보고, 그 사람만 베팅 버튼을 사용할 수 있습니다. 베팅 후 "베팅 확정"을 눌러야 다음 사람으로 넘어갑니다.
5. 한 바퀴를 다 돌면 다시 처음 사람부터 시작됩니다 (계속 베팅 라운드를 반복할 수 있음).
6. 카드 게임 결과가 나오면 방장이 **"승자 지정하기"**로 이긴 사람을 선택합니다. 팟 전액이 지급되고, 다음 판은 그 승자부터 턴이 시작됩니다.
7. 잔액이 0원이 된 사람은 자동으로 턴에서 빠지고 "파산" 표시가 붙습니다.

## 데이터 보존에 대해

게임 데이터는 서버 메모리에만 저장됩니다. 서버가 재시작되거나 모든 인원이 30분 이상 접속을 끊으면 해당 방의 데이터는 사라집니다. 한 게임이 끝나면 다시 쓰지 않는 용도에 맞춘 설계입니다.

새로고침을 하더라도 같은 브라우저 탭에서는 본인 정보(잔액, 턴 순서, 방장 권한)가 유지됩니다. 다만 브라우저를 완전히 새로 열거나 다른 기기로 접속하면 새 플레이어로 인식되니, 게임 중에는 본인이 입장했던 그 브라우저 탭을 계속 사용해주세요.
