# 토큰 카운터

친구들과 포커 칠 때 카드는 직접, 토큰(칩) 계산만 폰으로 실시간 공유하는 웹앱입니다.
방장이 방을 만들고 승자를 지정하는 방식으로 동작합니다.

## 동작 방식

- **방장**: "방 만들기"로 새 방 생성, 인당 시작 토큰 금액 설정. 본인도 플레이어로 참가해서 베팅 가능.
- **참가자**: 방 코드 입력 후 이름으로 입장. 모든 플레이어의 토큰 잔액이 화면에 보임.
- **베팅**: 자신의 카드에만 베팅 버튼(100 / 500 / 1000 / 올인)이 표시됩니다. 다른 사람의 카드에는 버튼이 없어서 남의 토큰을 조작할 수 없습니다. 버튼을 누르면 그 금액이 잔액에서 빠지고 상단 "총 베팅된 토큰(팟)"에 누적됩니다.
- **승자 지정**: 오프라인 카드 게임이 끝나면 방장만 "승자 지정하기"로 화면에서 한 명을 선택할 수 있고, 팟 전액이 그 사람에게 지급되고 팟은 0으로 초기화됩니다.

권한은 서버에서 검증합니다 — 본인 잔액이 아니면 베팅 자체가 거부되고, 방장이 아니면 승자 지정 요청이 거부됩니다.

## 로컬에서 실행

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속.

## Render에 배포하기

1. 이 폴더 전체를 GitHub 저장소에 올립니다.
2. [Render](https://render.com) 가입 후 **New + → Web Service** 선택.
3. 만든 GitHub 저장소를 연결합니다.
4. 설정값:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. **Create Web Service** 클릭 → 배포 완료되면 `https://your-app-name.onrender.com` 같은 주소가 생깁니다.

> 무료 플랜은 일정 시간 미사용 시 서버가 슬립 모드로 들어갑니다. 게임 시작 전 누군가 먼저 한 번 접속해서 서버를 깨워두면 좋습니다 (슬립 상태에서 첫 접속은 10~30초 정도 걸릴 수 있어요).

## 사용 방법

1. 방장이 **"방 만들기"** 탭에서 이름과 인당 시작 토큰(예: 1000)을 입력하고 방을 만듭니다.
2. 화면 하단의 **"링크 복사"**로 방 링크를 친구들에게 공유합니다 (또는 방 코드를 알려줘서 "방 입장" 탭에서 직접 입력하게 해도 됩니다).
3. 친구들은 링크를 열고 이름을 입력해 입장합니다. 입장하면 자동으로 시작 토큰만큼 잔액이 주어집니다.
4. 카드 게임은 오프라인에서 진행하고, 베팅할 때마다 각자 본인 화면에서 100/500/1000/올인 버튼으로 베팅합니다.
5. 게임이 끝나면 방장이 **"승자 지정하기"**를 눌러 이긴 사람을 선택합니다. 팟 전액이 그 사람에게 지급됩니다.
6. 다음 핸드도 같은 방식으로 반복합니다.

## 데이터 보존에 대해

게임 데이터는 서버 메모리에만 저장됩니다. 서버가 재시작되거나(예: 무료 플랜 슬립 후 재기동) 모든 인원이 30분 이상 접속을 끊으면 해당 방의 데이터는 사라집니다. 한 게임이 끝나면 다시 쓰지 않는 용도에 맞춘 설계입니다.

새로고침을 하더라도 같은 브라우저 탭에서는 본인 정보(잔액, 방장 권한)가 유지됩니다. 다만 브라우저를 완전히 새로 열거나 다른 기기로 접속하면 새 플레이어로 인식되니, 게임 중에는 본인이 입장했던 그 브라우저 탭을 계속 사용해주세요.
